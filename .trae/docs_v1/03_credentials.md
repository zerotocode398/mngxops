# MngxOps 需求文档 - 03 凭证管理

> **功能模块**: 凭证管理（Credentials）  
> **URL**: `/credentials/`  
> **模型**: `apps/credentials/models.py` → `Credential`, `CredentialEnableTask`  
> **视图**: `apps/credentials/views.py`

---

## 1. 功能概述

凭证管理模块负责统一管理所有远程节点的 SSH 认证信息，是整个系统的安全基石。包括：
- SSH 凭证的增删改查
- 密码认证与密钥认证双模式
- Fernet 加密存储敏感信息
- 凭证启用/禁用控制
- 凭证关联节点的批量连接测试
- 凭证使用情况追踪

## 2. 数据模型

### 2.1 Credential 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | CharField(100) | 凭证名称 |
| `username` | CharField(100) | SSH用户名 |
| `auth_type` | CharField(choices) | 认证方式：password/key |
| `password` | TextField(encrypted) | SSH密码（Fernet加密） |
| `private_key` | TextField(encrypted) | SSH私钥（Fernet加密） |
| `is_enabled` | BooleanField | 是否启用 |
| `description` | TextField | 描述 |
| `created_by` | FK → User | 创建人 |
| `created_at` | DateTimeField | 创建时间 |
| `updated_at` | DateTimeField | 更新时间 |

### 2.2 CredentialEnableTask 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `credential` | FK → Credential | 关联凭证 |
| `status` | CharField(choices) | 状态：pending/running/completed/failed |
| `total_count` | IntegerField | 总节点数 |
| `completed_count` | IntegerField | 已完成数 |
| `success_count` | IntegerField | 成功数 |
| `failed_count` | IntegerField | 失败数 |
| `skipped_count` | IntegerField | 跳过数 |
| `task_center_id` | BigIntegerField | 任务中心关联ID |
| `message` | TextField | 结果说明 |

- 保存成功后自动退出编辑模式

│  │ │ ✅ web02 (10.0.1.2)  - 连接成功           │  │  │
│  │ │ ❌ web03 (10.0.1.3)  - 认证失败           │  │  │
│  │ │ ...                                       │  │  │
│  │ └──────────────────────────────────────────┘  │  │

**业务逻辑**:
- 切换到"启用"状态时，自动触发关联节点的批量连接测试
- 测试通过异步任务方式执行（ThreadPoolExecutor）
- 在任务中心创建对应的 `TaskCenterTask` 记录
- 实时显示测试进度和每台节点的测试结果

## 5. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 加密存储 | 密码和私钥在保存前使用 Fernet 加密，密钥存储在 `utils/.fernet_key` |
| R2 | 安全提示 | 密码/私钥不显示在列表页；编辑页不回填原始值，仅提供"重新输入" |
| R3 | 禁用影响 | 禁用凭证后，关联节点的远程操作全部失败 |
| R4 | 启用测试 | 凭证从禁用切换为启用时，自动触发关联节点的批量连接测试 |
| R5 | 删除保护 | 删除凭证前检查是否有节点关联，有则提示确认并列出受影响的节点 |
| R6 | 唯一约束 | 同一用户下凭证名称必须唯一 |
| R7 | 认证方式 | 保存时验证：密码模式下密码必填，密钥模式下私钥必填 |
| R8 | 密钥格式 | 私钥保存前校验是否为合法的 RSA/DSA/ECDSA/Ed25519 格式 |

## 7. 重构建议

| 编号 | 问题 | 建议 |
|------|------|------|
| S1 | 密码/私钥编辑页无安全回填机制 | 编辑时字段留空，添加"使用当前密码/密钥"label提示 |
| S2 | 凭证测试结果未持久化 | 在 Credential 模型增加 `last_test_time` 和 `last_test_result` 字段 |
| S3 | 缺少凭证使用审计 | 在操作日志中记录凭证的新建/编辑/启用/禁用操作 |
| S4 | 凭证列表缺少预警 | 当凭证关联的节点大量离线时，在凭证列表标注警告 |
| S5 | 密钥类型不支持 Ed25519 | `utils/ssh.py` 中仅处理 RSAKey，需扩展支持 Ed25519Key / ECDSAKey |
