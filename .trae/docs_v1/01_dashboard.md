# MngxOps 需求文档 - 01 仪表盘

> **功能模块**: 仪表盘（Dashboard）  
> **URL**: `/`  
> **视图**: `apps/dashboard/views.py` → `index()`  
> **模板**: `apps/dashboard/templates/dashboard/index.html`

---

## 1. 功能概述

仪表盘是用户登录后的首页，提供系统整体运行状况的快速概览。核心定位是"一屏看全貌"，让运维人员快速掌握节点健康度与近期操作动态。

---

## 2. 数据模型

仪表盘不独立建模，聚合展示以下模型统计数据：

| 模型 | 统计指标 |
|------|---------|
| `Node` | 总节点数、在线数、离线数 |
| `NodeGroup` | 节点组总数 |
| `ConfigNodeBinding` | 待推送绑定数、冲突绑定数、下发失败的绑定列表 |
| `ReleaseTask` | 发布任务总数、最近发布任务列表 |

### 3.2 详细元素说明

#### 3.2.1 统计卡片区域

- **排列**: 响应式网格，大屏 4 列，中屏 3 列，小屏 2 列
- **样式**: Bootstrap Card + 图标 + 数字 + 标签
- **颜色语义**:
  - 在线节点卡片：绿色边框 / 绿色数字
  - 离线节点卡片：红色边框 / 红色数字
  - 待推送卡片：蓝色边框
  - 冲突卡片：橙色/黄色边框
  - 其他卡片：蓝色边框 / 默认色
- **交互**: 在线/离线/待推送/冲突卡片可点击跳转到对应列表页（带筛选条件）

**卡片指标说明**:

| 卡片 | 图标 | 数据来源 | 说明 |
|------|------|---------|------|
| 节点总数 | 📊 | `Node.objects.count()` | 所有节点 |
| 在线节点 | 🟢 | `Node.objects.filter(status="online").count()` | 最近一次探测状态为在线 |
| 离线节点 | 🔴 | `Node.objects.filter(status="offline").count()` | 最近一次探测状态为离线 |
| 节点组 | 📁 | `NodeGroup.objects.count()` | 节点组总数 |
| 发布任务 | 🚀 | `ReleaseTask.objects.count()` | 发布任务总数 |
| 待推送 | 📝 | `ConfigNodeBinding.objects.filter(sync_status="modified").count()` | 本地有修改但未推送到远程的绑定数 |
| 冲突 | ⚠️ | `ConfigNodeBinding.objects.filter(sync_status="conflict").count()` | 本地与远程均被修改、产生冲突的绑定数 |

> **注意**: 统计卡片不包含"配置版本"——按修正后的数据模型（`04_configs.md`），版本隶属于每条 `ConfigNodeBinding`（独立版本链），全局聚合版本总数无业务意义。也不包含"下发失败"——下方已有告警列表做详细展示，无需卡片级别重复。

#### 3.2.2 最近发布任务列表

| 列名 | 说明 |
|------|------|
| 批次号 | `task.batch_number` |
| 配置名称 | `task.config.name`，可点击跳转配置详情 |
| 版本 | `task.publish_version`（绑定版本号） |
| 目标节点 | `task.node.hostname` |
| 操作人 | `task.operator.username` |
| 状态 | 成功(绿)/失败(红)/进行中(蓝)/等待中(灰) 徽标 |
| 创建时间 | `task.created_at`，相对时间显示 |

#### 3.2.3 配置下发失败告警

展示所有 `ConfigNodeBinding.sync_status = "failed"` 的绑定记录，帮助运维人员快速发现并处理下发异常。

| 列名 | 说明 |
|------|------|
| 绑定 | `{config.name}@{node.hostname}`，可点击跳转绑定详情 |
| 关联节点 | `binding.node.hostname` |
| 下发状态 | "下发失败" 红色徽标 |
| 最后下发 | `binding.last_sync_time`，相对时间 |
| 错误信息 | `binding.last_sync_error`，截断显示(最多50字符)，hover 显示完整内容 |
| 操作 | "重试下发" 按钮 / "查看详情" 链接 |

## 4. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 定时刷新 | 统计卡片区域前端每 30 秒自动轮询轻量 API（仅查询统计数字，非完整页面刷新），轮询间隔可通过系统设置 `system.dashboard_refresh_interval` 配置；列表区域不自动刷新，提供"刷新"按钮手动刷新 |
| R2 | 轮询不触发 SSH | 在线/离线状态来自数据库中 `Node.status` 字段（由上次连接测试/发布操作更新），轮询仅刷新页面显示，不发起 SSH 探测 |
| R3 | 空状态 | 无数据时显示友好的空状态提示，如"暂无节点，去添加" |
| R4 | 权限控制 | 仅有对应模块 read 权限时显示相关卡片和列表 |
| R5 | 数量限制 | 最近发布任务列表和下发失败告警最多显示 10 条 |

## 5. 交互行为

| 交互 | 触发 | 行为 |
|------|------|------|
| 点击统计卡片 | 点击在线/离线/待推送/冲突数字卡片 | 跳转到对应列表页面，自动加上对应筛选条件 |
| 自动刷新 | 页面打开期间 | 前 30 秒轮询统计卡片 API，刷新数字（在线/离线/待推送/冲突计数），列表不自动刷新 |
| 点击刷新按钮 | 点击页面顶部刷新图标 | 重新加载全部页面数据（含列表） |
| Toast提示 | 登录后首次进入 | 显示欢迎消息 + 上次登录信息 |

## 7. 重构建议

| 编号 | 问题 | 建议 |
|------|------|------|
| S1 | 现有仪表盘仅显示原始数据，缺乏可视化 | 增加节点在线率环形图、同步状态饼图 |
| S2 | 统计数据无趋势对比 | 增加"较昨日"变化量（↑3 / ↓1） |
| S3 | 缺少快速操作入口 | 增加快捷操作按钮区（"添加节点""新建配置""发布"） |
| S4 | 告警列表操作不够直接 | 下发失败告警支持一键重试（单条/批量），减少跳转 |
