# MngxOps 需求文档 - 02 节点管理

> **功能模块**: 节点管理（Nodes）  
> **URL**: `/nodes/`  
> **模型**: `apps/nodes/models.py` → `Node`, `NodeGroup`  
> **视图**: `apps/nodes/views.py`

---

## 1. 功能概述

节点管理是 MngxOps 最核心的模块，负责管理所有 nginx 服务器节点的生命周期。包括：
- 节点的增删改查
- 节点组的创建与管理
- SSH 连接测试（单个/批量）
- 节点锁定/解锁
- 远程系统信息采集
- Nginx 版本自动检测
- 多维度搜索与筛选

---

## 2. 数据模型

### 2.1 Node 模型

| 字段            | 类型                          | 说明                         |
| --------------- | ----------------------------- | ---------------------------- |
| `hostname`      | CharField(100)                | 主机名                       |
| `ip`            | GenericIPAddressField(unique) | IP地址                       |
| `port`          | IntegerField(default=22)      | SSH端口                      |
| `groups`        | M2M → NodeGroup               | 所属节点组（最多3个）        |
| `credential`    | FK → Credential               | 关联SSH凭证                  |
| `environment`   | CharField(choices)            | 环境：dev/test/prod          |
| `nginx_version` | CharField(50)                 | nginx版本                    |
| `nginx_path`    | CharField(255)                | 自定义nginx路径              |
| `status`        | CharField(choices)            | 状态：online/offline/unknown |
| `is_locked`     | BooleanField                  | 是否锁定                     |
| `description`   | TextField                     | 描述                         |
| `created_by`    | FK → User                     | 创建人                       |
| `created_at`    | DateTimeField                 | 创建时间                     |
| `updated_at`    | DateTimeField                 | 更新时间                     |

### 2.2 NodeGroup 模型

| 字段          | 类型                   | 说明     |
| ------------- | ---------------------- | -------- |
| `name`        | CharField(100, unique) | 组名     |
| `description` | TextField              | 描述     |
| `created_by`  | FK → User              | 创建人   |
| `created_at`  | DateTimeField          | 创建时间 |

#### 3.1.2 搜索与筛选

所有筛选条件变更后**自动触发查询**，无需手动点击搜索按钮：

| 控件类型 | 触发方式 | 防抖策略 |
|---------|---------|---------|
| 搜索输入框（主机名/IP） | 输入后自动查询 | **300ms 防抖**（避免每次击键都发起请求） |
| 节点组下拉（多选） | 勾选/取消后立即查询 | 无防抖 |
| 环境下拉（单选） | 切换后立即查询 | 无防抖 |
| 状态下拉（单选） | 切换后立即查询 | 无防抖 |
| 每页条数 | 切换后立即查询 | 无防抖 |

- **搜索输入框**: 支持主机名/IP模糊搜索，支持多关键字（逗号分隔），支持智能语义（如"在线""离线""生产"自动转为筛选条件）
- **节点组筛选**: 下拉多选
- **环境筛选**: 下拉单选：全部/开发/测试/生产
- **状态筛选**: 下拉单选：全部/在线/离线/未知
- **每页条数**: 10/20/50/100 可选
- **记住搜索状态**: 通过 sessionStorage 保存搜索条件，返回页面时恢复
- **自动查询时机**: 页面首次加载时应用已保存的筛选条件并自动查询；筛选条件变更时自动重新查询

### 3.2 节点详情弹窗

采用**两栏横屏布局**，左侧基本信息 + 操作记录，右侧 Nginx/系统信息，避免竖屏信息过长时篇幅冗长。

```
┌──────────────────────────────────────────────────────────────────┐
│  📡 web01 详情                                                   │
│                                                                  │
│  ┌─ 左栏：基本信息 ─────────────┐ ┌─ 右栏：服务信息 ──────────┐ │
│  │                              │ │                            │ │
│  │ 主机名     web01             │ │ 🛠 Nginx 信息               │ │
│  │ IP地址     10.0.1.1          │ │   版本   nginx/1.26.1      │ │
│  │ SSH端口    22                │ │   路径   /usr/local/nginx  │ │
│  │ 环境       🏭 生产环境       │ │   [🔄 重新检测版本]        │ │
│  │ 状态       🟢 在线           │ │                            │ │
│  │ 锁定       🔓 否             │ │ 🖥 系统信息                 │ │
│  │ 凭证       prod-key(testuser)│ │   OS     Ubuntu 22.04      │ │
│  │ 节点组     prod-web, core    │ │   Kernel 5.15.0            │ │
│  │ 描述       生产Web服务器主节点│ │   CPU    Xeon 2.5GHz×8     │ │
│  │                              │ │   Memory 16G (Used 8.5G)   │ │
│  │ ──────────────────────────── │ │   Disk   100G (Used 45G)   │ │
│  │ 📅 操作记录                   │ │   Uptime 30 days, 5h       │ │
│  │ 创建人     admin             │ │   [🔄 刷新系统信息]        │ │
│  │ 创建时间   2026-06-01 10:00  │ │                            │ │
│  │ 更新时间   2026-07-10 08:30  │ │                            │ │
│  └──────────────────────────────┘ └────────────────────────────┘ │
│                                                                  │
│  [编辑]  [🔌 测试连接]  [🔒 锁定]  [关闭]                         │
└──────────────────────────────────────────────────────────────────┘
```

**布局规范**:
- 弹窗宽度：**800px**（桌面）/ 95vw（移动端）
- 左右两栏比例：`flex: 1`，中间竖线分隔
- 移动端（<768px）：两栏自动折叠为单栏上下布局
- 系统信息与 Nginx 信息共用右栏

**数据加载策略**:
- 基本信息、操作记录：点击弹窗时从 DOM 或缓存立即展示（无网络请求）
- Nginx 信息、系统信息：弹窗打开后异步加载（Ajax），避免阻塞弹窗打开；未加载完成时右侧显示骨架屏占位

**交互要求**:
- 弹窗居中显示，背景遮罩层半透明
- 点击"编辑"跳转到编辑页面
- 点击"测试连接"：**单个节点连接测试为同步 Ajax 操作**（不走异步任务 TaskCenterTask），弹窗内按钮显示 loading 状态，10 秒超时后返回结果，成功后自动更新节点在线状态和 nginx 版本字段
- 点击"重新检测版本"：通过 `/nodes/nginx-version/` Ajax 获取，按钮 loading 状态
- 点击"刷新系统信息"：通过 `/nodes/system-info/` Ajax 获取，按钮 loading 状态
- 系统信息区域内容较长时支持内部滚动（`max-height: 200px; overflow-y: auto`）

## 5. 业务规则

| 编号 | 规则       | 说明                                                  |
| ---- | ---------- | ----------------------------------------------------- |
| R1   | 节点组上限 | 每个节点最多属于 3 个节点组                           |
| R2   | 锁定保护   | 锁定节点禁止远程操作（测试连接/发布），但可编辑和删除 |
| R3   | 批量限制   | 批量测试/锁定/解锁一次最多操作 3 个节点               |
| R4   | 连接测试   | 测试时自动更新节点状态和 nginx 版本；**单个节点测试为同步 Ajax（10s 超时），批量测试走异步任务 TaskCenterTask** |
| R5   | 凭证依赖   | 仅关联已启用的凭证时才能进行远程操作                  |
| R6   | IP唯一性   | IP地址全局唯一（模型层 unique 约束）                  |
| R7   | 删除约束   | 删除节点前检查是否有配置关联，有则提示确认            |
| R8   | 并发安全   | 批量异步任务使用数据库字段跟踪状态                    |
| R9   | 搜索自动查询 | 所有筛选条件变更后自动触发查询，输入框 300ms 防抖，下拉立即查询 |
| R10  | 凭证下拉分组 | SSH 凭证下拉按认证方式分组：🔒 密码认证在前，🔑 密钥认证在后，仅显示已启用的凭证 |

## 7. 重构建议

| 编号 | 问题                                     | 建议                                                       |
