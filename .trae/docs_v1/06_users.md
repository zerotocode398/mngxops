# MngxOps 需求文档 - 06 用户与权限

> **功能模块**: 用户与权限（Users & Permissions）  
> **URL**: `/users/`  
> **模型**: `apps/users/models.py` → `UserProfile`, `UserGroup`, `UserTeam`, `PermissionItem`  
> **视图**: `apps/users/views.py`

---

## 1. 功能概述

用户与权限模块负责平台的用户管理和权限控制，实现细粒度的 RBAC（基于角色的访问控制）。包括：
- 用户管理：CRUD、锁定/解锁、个人资料
- 角色管理：创建角色、分配权限
- 用户组管理：团队分组、批量授权
- 权限体系：资源 × 动作（read/create/update/delete）
- 登录认证与密码管理

## 2. 数据模型

### 2.1 User（Django 内置）

| 字段 | 说明 |
|------|------|
| `username` | 用户名（唯一） |
| `password` | 密码（哈希存储） |
| `is_active` | 是否激活（锁定/解锁） |
| `is_staff` | 是否可访问 admin |
| `is_superuser` | 超级管理员 |

### 2.2 UserProfile 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `user` | OneToOne → User | 关联用户 |
| `avatar` | ImageField | 头像 |
| `groups` | M2M → UserGroup | 角色（多角色） |
| `direct_permissions` | M2M → PermissionItem | 用户直授权限 |
| `remark` | TextField | 备注 |

### 2.3 UserGroup（角色） 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | CharField(100, unique) | 角色名称 |
| `description` | TextField | 描述 |
| `permissions` | M2M → PermissionItem | 角色权限 |
| `created_by` | FK → User | 创建人 |
| `created_at` | DateTimeField | 创建时间 |
| `updated_at` | DateTimeField | 更新时间 |

### 2.4 UserTeam（用户组） 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | CharField(100, unique) | 组名 |
| `description` | TextField | 描述 |
| `members` | M2M → User | 组成员 |
| `roles` | M2M → UserGroup | 关联角色 |
| `created_by` | FK → User | 创建人 |

### 2.5 PermissionItem 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | CharField(100, unique) | 权限编码（如 `nodes.read`） |
| `name` | CharField(100) | 权限名称（如 "节点查看"） |
| `resource` | CharField(50) | 资源（nodes/credentials/configs/releases/users/roles/teams/audit） |
| `action` | CharField(20) | 动作（read/create/update/delete） |

### 2.6 权限体系

```
权限来源层级（优先级从高到低）：
1. is_superuser → 拥有所有权限（最高优先级）
2. 用户直授权限 (direct_permissions)
3. 用户角色权限 (groups → permissions)
4. 用户所在用户组的角色权限 (user_teams → roles → permissions)
```

**权限矩阵交互**:
- 资源×动作的网格布局，清晰直观

- 不可用的权限项显示 `-`（如审计的 create/update/delete）

## 4. API接口

| URL | 方法 | 说明 |
|-----|------|------|
| `/users/` | GET | 用户列表 |
| `/users/create/` | GET/POST | 创建用户 |
| `/users/<id>/edit/` | GET/POST | 编辑用户 |
| `/users/<id>/delete/` | POST | 删除用户 |
| `/users/<id>/toggle/` | POST | 锁定/解锁用户 (Ajax) |

| `/users/roles/create/` | GET/POST | 创建角色 |
| `/users/roles/<id>/edit/` | GET/POST | 编辑角色 |
| `/users/roles/<id>/delete/` | POST | 删除角色 |
| `/users/teams/` | GET | 用户组列表 |
| `/users/teams/create/` | GET/POST | 创建用户组 |
| `/users/teams/<id>/edit/` | GET/POST | 编辑用户组 |
| `/users/teams/<id>/delete/` | POST | 删除用户组 |
| `/accounts/profile/` | GET | 个人中心 |
| `/accounts/password-change/` | GET/POST | 修改密码 |
| `/accounts/login/` | GET/POST | 登录 |
| `/accounts/logout/` | GET | 登出 |
| `/users/api/list/` | GET | 用户列表 API (JSON) |

## 5. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 权限合并 | 用户的最终权限 = 直授权限 ∪ 角色权限 ∪ 所属用户组的角色权限（取并集） |
| R2 | 超级管理员 | is_superuser=True 的用户拥有所有权限，不受权限配置影响 |
| R3 | 自我锁定 | 不能锁定自己的账号 |
| R4 | 密码安全 | 创建用户时密码必填（≥8位，含大小写+数字）；编辑用户时留空则不修改 |
| R5 | 角色删除 | 删除角色前检查是否有用户关联，有则提示 |
| R6 | 唯一约束 | 用户名全局唯一，角色名全局唯一，用户组名全局唯一 |
| R7 | 权限模型 | 用户-角色 多对多，用户-直授权限 多对多，角色-权限 多对多 |
| R8 | 登录限制 | 用户登录失败5次后临时锁定30分钟 |

## 7. 重构建议

| 编号 | 问题 | 建议 |
|------|------|------|
| S1 | 用户权限查询效率低 | 增加权限缓存（Django cache），避免每次请求都查询多表 |
| S2 | 权限矩阵 UI 可读性差 | 改用分组折叠面板 + 权限树形选择 |
| S3 | 缺少权限变更审计 | 角色/权限变更时记录审计日志 |
| S4 | UserViewSet 逻辑分散 | 统一使用 Django Admin 或 DRF ViewSet 管理 |
| S5 | 个人中心功能单一 | 增加"我的操作日志""我的登录记录""操作统计" |
| S6 | 用户组创建/编辑缺少批量操作 | 支持从文件（CSV）批量导入用户 |
