# MngxOps 需求文档 - 07 审计日志

> **功能模块**: 审计日志（Audit）  
> **URL**: `/audit/`  
> **模型**: `apps/audit/models.py` → `AuditLog`, `LoginLog`  
> **视图**: `apps/audit/views.py`

---

## 1. 功能概述

审计日志模块负责记录系统中所有关键操作和登录行为，为安全审计、问题追溯、合规性检查提供数据支撑。包括：
- 操作日志：记录所有用户的关键业务操作
- 登录日志：记录所有用户的登录行为（成功/失败）
- 日志筛选与搜索
- 日志详情查看（含操作上下文）

---

## 2. 数据模型

### 2.1 AuditLog 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `user` | FK → User | 操作用户 |
| `module` | CharField(100) | 操作模块（如"节点管理""配置管理"） |
| `action` | CharField(255) | 动作描述（如"创建节点""删除配置"） |
| `ip` | CharField(50) | 操作者IP地址 |
| `result` | CharField(choices) | 结果：success / failed |
| `detail` | TextField | 详情描述 |
| `created_at` | DateTimeField | 创建时间 |

### 2.2 LoginLog 模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `username` | CharField(150) | 尝试登录的用户名 |
| `ip` | CharField(50) | 登录IP地址 |
| `user_agent` | TextField | 浏览器 User-Agent |
| `status` | CharField(choices) | 结果：success / failed |
| `created_at` | DateTimeField | 登录时间 |

#### 3.1.1 表格列说明

| 列 | 内容 | 交互 |
|----|------|------|
| 时间 | `created_at`，精确到秒 | - |
| 操作用户 | `user.username` | 点击查看用户详情 |
| 模块 | `module`（节点管理/配置管理/发布管理/凭证管理/用户管理/登录管理/系统设置） | 筛选 |
| 动作 | `action`，如"创建 web01""编辑 nginx.conf""发布配置 V12→web01" | hover 显示完整详情 |
| IP | `ip` | - |
| 结果 | ✅ 成功 / ❌ 失败 徽标 | 筛选 |

#### 3.1.2 搜索与筛选

- **搜索输入框**: 支持模块名、动作描述、用户名模糊搜索
- **操作模块筛选**: 全部/节点管理/凭证管理/配置管理/发布管理/用户管理/登录管理
- **结果筛选**: 全部/成功/失败
- **日期范围**: 开始日期 - 结束日期（日期选择器）
- **操作人筛选**: 下拉选择用户
- **每页条数**: 10/20/50/100

## 5. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 日志记录 | 所有 CRUD 操作和远程操作均需记录 AuditLog |
| R2 | 日志不可改 | 审计日志只能查看，不能编辑或删除（合规要求） |
| R3 | 日志保留 | 操作日志保留至少 1 年（建议 3 年），登录日志保留 6 个月 |
| R4 | 登录失败记录 | 无论用户是否存在，登录失败都记录 LoginLog |
| R5 | 登录成功记录 | 登录成功记录 LoginLog + AuditLog 各一条 |
| R6 | IP 获取 | 优先使用 X-Forwarded-For 头，其次 REMOTE_ADDR |
| R7 | 敏感脱敏 | 日志详情中不记录密码、私钥等敏感信息 |
| R8 | 分页默认 | 按时间倒序排列，默认每页 20 条 |

## 7. 重构建议

| 编号 | 问题 | 建议 |
|------|------|------|
| S1 | 现有日志记录分散在各视图，缺乏统一拦截 | 使用 Django Signal + 统一日志服务层，减少重复代码 |
| S2 | 登录日志无统计分析 | 增加登录统计面板（今日登录数/失败数/活跃用户数） |
| S3 | 日志详情存储不够结构化 | 使用 JSONField 存储结构化操作上下文 |
| S4 | 缺少日志导出功能 | 支持导出为 CSV/Excel 格式 |
| S5 | 缺少异常检测 | 增加异常登录检测（异地IP、非常用设备）告警 |
| S6 | 登录日志失败原因不明确 | 区分"用户不存在""密码错误""用户已锁定"等失败原因 |
