#### 8.1 搜索筛选机制逐模块对比（核心不一致问题）

| 模块 | 搜索输入 | 触发方式 | 筛选下拉 | 防抖 | 记住搜索 |
|------|---------|---------|---------|------|---------|
| **节点管理** | tag-input（标签输入框+回车） | 回车提交 FORM | 无（无环境/状态/节点组下拉） | ❌ 无 | URL 参数 |
| **配置管理** | tag-input（标签输入框+回车/blur） | 回车或失焦提交 FORM | 仅状态（在线/离线/未知，对配置无意义） | ❌ 无 | sessionStorage |
| **凭证管理** | tag-input（标签输入框+回车） | 回车提交 FORM | 无（无认证方式/状态下拉） | ❌ 无 | URL 参数 |
| **发布历史** | 普通 input + 搜索按钮 | 点击搜索按钮 | 状态下拉 | ❌ 无 | sessionStorage |
| **审计日志** | 普通 input + 搜索按钮 | 点击搜索按钮 | 模块/操作类型下拉+日期 | ❌ 无 | URL 参数 |

**需求文档要求（08_ui_style_guide.md §3.8 + 各模块文档）：**
- 所有列表页统一的搜索筛选栏：`d-flex gap-2 align-items-center flex-wrap`
- 搜索输入框：300ms 防抖，输入后自动查询
- 下拉筛选：切换后立即查询
- "每页条数"统一在最右侧

**与需求差距**：
- ❌ 5 个模块用了 4 种不同的搜索机制（tag-input 回车 / tag-input 回车+blur / input+按钮 / input+按钮+日期）
- ❌ 全部无防抖（需求要求 300ms）
- ❌ 节点/凭证缺少环境/状态下拉筛选
- ❌ 发布/审计需手动点击搜索按钮（需求要求自动查询）

## 四、改造策略

### 4.1 推荐方案：分 4 个批次

```
批次1：基础设施层（铺路基）— 9.5 天 — 🟢风险低
├── 09 系统设置模块（模型+页面+setting_service）
├── 08 UI 统一：Toast + Modal + 侧边栏折叠
├── 08 UI 统一：CSS 独立 + 相对时间 + 空状态组件
└── 硬编码参数迁移（12 处 → get_setting()）

批次2：核心模型重构（动骨架）— 27 天 — 🔴风险高
├── 04 配置管理：模型+数据迁移+视图+模板重构（8.5天）
├── 05 发布管理：模型+视图+模板重构（6.5天）
├── 01 仪表盘：增强完善（2天）
├── 08 UI 搜索/表格/分页统一（5.5天）
└── 02 节点管理：视图拆分+详情弹窗+搜索防抖（4.5天）

批次3：原有模块增强（修细节）— 5 天 — 🟢风险低
├── 03 凭证增强：密钥类型扩展+搜索筛选+Toggle+显示/隐藏
└── 07 审计增强：JSONField+日志导出+详情弹窗+异常标记

批次4：全新功能模块（添新翼）— 9 天 — 🟡风险中
├── 10 Nginx 编译升级（模型+页面+SSH 流程）
└── 06 用户权限 UI 优化（权限矩阵+缓存）
```

### 4.2 分批理由

**批次1 先行（铺路基）**：
- 系统设置是全新模块，不修改现有代码，零风险
- Toast/Modal 组件建好后，后续所有模块可直接统一使用
- 侧边栏折叠好之后，新建"运维工具"分组可直接容纳 Nginx 升级模块
- setting_service 建好后，批次2 改造时可直接替换硬编码参数
- **这是后续所有工作的基础设施，必须最先完成**

**批次2 核心重构（动骨架）**：
- Config 模型重构是整个需求文档最核心的设计变更
- 发布模块强依赖配置模块，必须配套改造
- 在 UI 组件（批1已完成）的基础上做统一搜索/表格/分页
- **风险最高，建议单独拉分支开发，合并前充分测试**

**批次3 细节增强（修细节）**：
- 改动量小，不修改核心逻辑
- 可以独立开发，不受其他模块影响

**批次4 全新功能（添新翼）**：
- Nginx 升级是全新模块，虽然代码量最大但完全独立
- 可以并行开发（在另一个分支），不影响主线

## 五、风险提示

| 风险 | 等级 | 影响 | 缓解措施 |
|------|------|------|---------|
| Config 模型重构导致数据丢失 | 🔴高 | 配置内容/版本历史全部丢失 | 严格按迁移路径：新建模型→迁移脚本→废弃旧字段→删除旧表 |
| ReleaseTask 版本引用断裂 | 🔴高 | 所有历史发布记录无法关联 | 迁移时建立 ConfigVersion → BindingVersion 映射关系 |
| 视图文件合并冲突 | 🟡中 | 多模块同时改造冲突 | 分批进行，每个批次独立分支，合入前 rebase |
| SSH 密钥类型扩展影响现有连接 | 🟡中 | 已有 RSAKey 节点连接失败 | 保留原有 RSA 逻辑，新增 Ed25519/ECDSA 分支 |
| Nginx 升级编译环境差异 | 🟡中 | 目标节点缺少 gcc/make 等依赖 | 编译前预检依赖并给出安装提示 |
| UI 组件替换后样式不一致 | 🟢低 | 个别页面视觉异常 | 渐进替换，保留 Bootstrap 原始组件兼容 |
