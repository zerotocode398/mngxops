09-节点管理设计.md
```shell
1. 问题1 - 新增节点时是否强制测试 SSH
    需要进行手动测试，若未手动测试则自定义弹窗提醒。
    同时节点列表里也需要体现心跳检测按钮触发心跳检测，触发心跳检测，检测结果会显示在节点列表里。

2. 问题2 - 节点删除是否允许物理删除
    统一逻辑删除，数据库保留审计记录。

3. 问题3 - 一个节点是否允许管理多个 Nginx 实例。
    不允许，通常都是一个节点管理一个 Nginx 实例。

4. 问题4 - nginx 非默认路径
    nginx 有自定义编译场景，存在需要手动明确 nignx 可执行二进制文件路径的情况。

5. 问题5 - 节点和节点组
    节点组、节点都可以关联凭证，当两者同时关联凭证时优先使用节点的凭证。
    若节点未关联凭证，会使用节点组的凭证。
    若节点组未关联凭证，会使用默认凭证。

6. 问题6 - 节点展示
    默认展示10台节点，用户可以根据需要调整展示数量。
    节点列表里也会体现节点组、节点、凭证等关联关系。
```

12-配置管理设计.md
```shell
1. 问题1 - 管理的到底是什么配置？
    nginx 配置，包括但不限于：nginx.conf 以及 nginx.conf 里 include 的所有配置文件（不包括 mime.types及内置的配置文件）。

2. 问题2 - 配置是存文件还是存数据库？
    配置是存数据库，数据库里会保存配置文件的内容。

3. 问题3 - 发布策略
    发布前需自定义窗口提示，确认是否发布？
    发布可以批量发布，也可以单台发布。
    发布的时候可以选择发布到指定的节点组、节点。
    发布的时候可以选择一个配置或者多个配置文件。
    其流程可以是，点击发布 -> 弹窗确认 -> 配置发布目标 -> nginx -t 检测 -> 发布，若失败终止发布

4. 问题4 - Reload由谁执行？
    自动，但是需要弹窗确认。
    但是用户也可以手动触发，点击Reload按钮，无需弹窗确认。

5. 问题5 - 是否允许在线编辑？
    不允许，只允许编辑本地的配置文件。
    编辑完成后，发布到节点时，会自动将本地的配置文件上传到节点。
    发布后，在决定是否 Reload。
6. 问题6 - 是否增加“配置模板管理”模块？
    目前不需要，目前是管理多台 nginx 的配置文件，而不是配置模板。

7. 问题7 - 配置获取
    支持手动导入或者自动拉取，用户需要提供 nginx.conf 的路径，自动解析 include 的所有配置文件。不包括 mime.types及内置的配置文件。
    支持批量更新或者单台更新。

8. 问题8 - 配置历史版本
    配置只要每次更新，就需要单独记录一个历史版本，比如 YYYYMMDD - V(版本号)。
    历史版本页面默认展示最近10个版本，用户可以根据需要调整展示数量，最多支持保存近 30 天的历史版本。

9. 问题9 - 最大的问题：配置归属于谁？
    配置归属与节点，1 对 1 关联。即使多台 nginx 实例，每个实例的配置文件也是独立的。因为配置是从目标机器或者手动同步的。
    配置文件可以是 nginx.conf，也可以是 include 的所有配置文件。

10. 问题10 - 自动拉取需要增加“配置来源”
    需要

11. 问题11 - 发布前 nginx -t 检测位置有问题
    可以，以你的为准

12. 问题12 - Reload 应该拆出来
    可以，以你的为准

13. 问题13 - 历史版本建议改成永久保留 + 自动清理
    可以，以你的为准

14. 问题14 - 是否增加配置集
    可以，以你的为准

15. 问题14 - 配置产生差异
    发现差异后提示用户确认
    发现差异 -> 生成变更列表 -> 用户确认 -> 更新数据库
```

13. 配置发布设计
```shell
1. 问题1 - 发布是否需要审批？
    通过 RBAC 权限控制

2. 问题2 - 发布失败策略
    继续执行

3. 问题3 - nginx -t 在哪里执行？
    目标机器

4. 问题4 - 回滚粒度
    重新发布旧版本，可以选择节点全量配置或某个配置。

5. 问题5 - 发布是否支持定时？
    否

6. 问题6 - 节点选择方式
    配置版本
      ↓
节点组
      ↓
节点列表
      ↓
勾选节点
      ↓
发布

7. 问题7 - 发布方式
    数据库
      ↓
SSH
      ↓
目标节点

8. 问题8 - 配置文件覆盖策略
    目标目录完全同步

9. 问题9 - 发布单位是什么？
    全量发布、单配置发布

10. 问题10 - Nginx 配置路径是否统一？
    否
    不同节点的配置路径可能不同，需要根据节点的配置路径进行发布。

11. 问题11 - 发布后是否做健康检查？
    不做，但是可以做 nginx 进程检查。

12. 问题12 - 发布目录同步方式
    镜像同步
```
